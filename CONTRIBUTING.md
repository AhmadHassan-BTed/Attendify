# Contributing

Thank you for your interest in contributing to this university attendance portal project.

## Contribution guidelines

- Open an issue for bugs, enhancements, or questions.
- Use a clear title and describe the expected behavior.
- Submit pull requests with a concise description, changes, and testing steps.
- Keep changes focused and avoid modifying unrelated files.

## Development workflow

1. Fork the repository.
2. Create a branch with a descriptive name.
3. Make your changes and test locally.
4. Submit a pull request to the main branch.

## Notes

- If you add new dependencies, explain why in the PR description.
- For backend changes, verify the API behavior and database compatibility.
- For frontend changes, verify the UI flows and form validations.
